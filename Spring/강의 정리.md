# 스프링 강의

## 3강 프로젝트 생성

1. 스프링부트 스타터 사이트에서 스프링 프로젝트 생성: 
start.spring.io

프로젝트 Maven vs Gradle: 필요한 라이브러리를 가져오고 빌드하는 라이프사이클을 관리하는 툴. 과거에는 Maven, 요즘은 Gradle을 많이 쓰는 추세. 레거시 프로젝트를 제외한 요즘 실무 프로젝트는 Gradle로 많이 함.

Spring Boot 
- snapshot: 만들고 있는 거
- M1 -> 정식 release 된 게 아님

Group: 보통 기업 도메인 명. 지금은 아무거나 hello
Artifact: 빌드되어 나온 결과물: hello-spring
Name, Description: 그냥 그대로 하셈

Dependency: 우리 프로젝트에서 어떤 라이브러리 쓸 건지.
웹프로젝트 -> Spring Web
thymeleaf(타임리프) -> html을 만들어주는 템플릿 엔진. 근데 회사마다 쓰는 게 다르다.

다운로드하면 zip 파일 생기는데, 압축 풀어서 build.gradle 선택해서 열어주면 됨. 

.idea -> intelij가 관리하는 설정파일
.gradle (그레이들)
src에 main과 테스트라는 폴더가 나누어져있다.
resources -> xml, properties, html 등 자바 파일을 제외한 나머지 파일들.

build.gradle -> 옛날에는 이 파일을 다 짰어야했지만, 스프링부트가 나오면서 이런 설정 파일들이 생성된다. 
여기 안에 dependencies에 아까 선택햇던 것들이 들어감. 그리고 Junit 5 자동으로 들어감.

mavenCentral()이라는 사이트에서 이런 라이브러리들 다운받아라 하는 거.

.gitignore -> git에는 필요한 소스코드만 올라가야하고, 빌드된 결과물은 올라가면 안됨. 

gradlew -> 빌드 관련
gradlew.bat -> 빌드 관련
setting.gradle -> gradle 배우면 알 수 있는 것.


src.main.java 안에
package hello.hellospring 있음
여기에 @SpringBootApplication 어노테이션이 있다.
main 메서드 실행하면, Tomcat started on 포트 8080 http
http//localhost:8080 -> 404 뜸. 그럼 성공.
톰캣이라는 웹서버 내장. 실행하면 자체적으로 띄우면서 springboot가 올라온다.

설정에서 gradle 검색 후 gradle -> intelij 로 바꿔서 빠르게 띄울 수 있다.
