# 스프링 강의

## 3강 프로젝트 생성

1. 스프링부트 스타터 사이트에서 스프링 프로젝트 생성하기 
start.spring.io 사이트

(1) 프로젝트 Maven vs Gradle: 필요한 라이브러리를 가져오고 빌드하는 라이프사이클을 관리하는 툴. 과거에는 Maven, 요즘은 Gradle을 많이 쓰는 추세. 레거시 프로젝트를 제외한 요즘 실무 프로젝트는 Gradle로 많이 함.

(2) Spring Boot 버전
- snapshot: 만들고 있는 거
- M1 -> 정식 release 된 게 아님

(3) 나머지
Group: 보통 기업 도메인 명. 지금은 아무거나 hello
Artifact: 빌드되어 나온 결과물: hello-spring
Name, Description: 그냥 그대로 하셈

(4) Dependency
Dependency: 우리 프로젝트에서 어떤 라이브러리 쓸 건지.
- 웹프로젝트 -> Spring Web
- thymeleaf(타임리프) -> html을 만들어주는 템플릿 엔진. 근데 회사마다 쓰는 게 다르다.

다운로드하면 zip 파일 생기는데, 압축 풀어서 build.gradle 선택해서 열어주면 됨. 

2. 프로젝트 파일들
.idea -> intelij가 관리하는 설정파일
.gradle (그레이들)

src에 main과 테스트라는 폴더가 나누어져있다.
resources -> xml, properties, html 등 자바 파일을 제외한 나머지 파일들.

build.gradle -> 옛날에는 이 파일을 다 짰어야했지만, 스프링부트가 나오면서 이런 설정 파일들이 생성된다. 
여기 안에 dependencies에 아까 선택했던 것들이 들어감. 그리고 Junit 5 자동으로 들어감.
그 안에 mavenCentral()는 이 사이트에서 이런 라이브러리들 다운받아라 하는 거.

.gitignore -> git에는 필요한 소스코드만 올라가야하고, 빌드된 결과물은 올라가면 안됨. 

gradlew -> 빌드 관련
gradlew.bat -> 빌드 관련
setting.gradle -> gradle 배우면 알 수 있는 것.

## 4강 라이브러리 
1. build.gradle에서 우리는 타임리프(html 만들어주는 템플릿 엔진), 웹, test 3개

2. Externel Libraries에 라이브러리가 엄청 많다. 아파치, 톰캣, 제이유닛, 스프링부트 스타터 로깅, 코어, 등등 엄청 많다.

maven이나 gradle은 의존관계를 관리. springboot-starter-web 라이브러리는 톰캣, 스프링 웹, 스프링 코어 등의 라이브러리들이 필요하다. gradle은 내가 필요한 라이브러리가 필요한 라이브러리들을 같이 가져와준다. 

3. 중요 라이브러리들
alt 두번 -> 오른쪽에 gradle 뜬다. 거기에 hello-spring.Dependencies 를 열어보면 라이브러리들 의존관계를 볼 수 있다.

spring-boot-starter-web 안에 spring-boot-starter-tomcat이 있다. 옛날에는 서버에 was를 설치 후 자바 코드를 집어넣는 식으로 개발. 웹서버와 개발 라이브러리들이 분리되어있었음. 요즘에는 소스 라이브러리에서 웹서버를 내장, 임베디드하고 있다. java 메인 메서드만 실행해도 8080 포트에 웹서버 뜬다. 
spring-boot-starter에 spring-core 라이브러리 중요. 

log: 현업은 System.out.println으로 출력하면 안됨. 로그로 출력을 해야한다. 로그로 남겨야 심각한 에러만 볼 수 있고, 로그파일들 관리 가능. 서버 세상을로 안넘어온 사람들은 로그 왜 쓰지라는 생각이 들 수 있다. spring-boot-starter에 starter-logging에, slf4j 인터페이스. 요즘은 logback을 많이 쓴다. logback과 slf4j를 두개를 동시에 쓴다.

test 관련 라이브러리. starter-test 안에 junit 라이브러리 많이 씀. 4가 오래 유지되다가 5로 최근 넘어감. 
mokito, assertj는 테스트를 편리하게 하게 도와주는 것.
spring-test -> 스프링과 통합을 해서 테스트하는 라이브러리
junit이 핵심

정리하면, 
spring-boot-starter-web 안의 "톰캣" 과 "spring-webmvc"
spring-boot-starter-thymeleaf(타임리프 템플릿 엔진(view))
spring-boot-starter: 스프링부트 + 코어 + 로깅
로깅은 (logbakc,slf4j)
테스트 라이브러리 junit, mockito, assertj, spring-test
