## 클래스
1. 클래스
설계도, 설명서 역할. 데이터와 기능을 포함하는 묶음.
이 클래스를 사용해서 여러가지 객체를 만들어낼 수 있음.
마치 종이학 접기 책(클래스)를 바탕으로, 종이학을 접어낸 것(객체)과 같음.

`Person person = new Person();`
Person 클래스로부터 새로운 객체 person을 만들어낸 것.

2. 인스턴스 변수
클래스 내 선언된 변수.
클래스 명 뒤 중괄호 안에 정의하는 변수
```java
// class 정의
class Person{
    String name; // -> 이 name과 age가 인스턴스 변수
    int age;
}
```

```java
// 객체 생성 및 인스턴스 변수 접근
public static void main(String[] args){
  Person person = new Person();
  person.name = "이승희" // 인스턴스 변수
  person.age = 26;
}
```
3. 클래스 변수
static으로 정의되는, 모든 클래스가 공유하는 변수. (사실 이건 객체를 생성하지 않고도 사용하는 변수라고 정의하는 게 더 좋을 듯)
```java
class Person{
  String name;
  int age;
  int personCount; // 클래스 변수
```

```java
// 이렇게 객체를 생성하지 않아도 클래스 명으로 접근 가능.
public static void main(String[] args){
  Person.personCount = 10;
  System.out.println(Person.personCount); //10
}
```
언제 쓰고, 언제 안쓸지 생각해볼 지점.

5. 인스턴스 메소드
클래스 내에 정의된 메소드.
접근은, 객체를 생성하고, .메소드() 로 쓰면 된다.

6. 클래스 메소드
클래스 내에 static으로 정의된 메소드.
위처럼 객체를 생성하지 않고도 사용가능하겠죠.

7. This
자기 자신을 의미. 인스턴스랑 지역변수를 구분하기위해서 사용한다.
예를 들어
```java
class Person {
  String name;
  public void setName(String name){
    this.name = name; // 여기서 this.name 은 위에 정의한 String name을 의미하는 것이고, 할당자 뒤 name은 파라미터로 받은 name 값을 의미하는 것이다.
}
```

8. 생성자
객체가 생성될 때 호출되는 메소드. 객체가 생성되자마자 변수를 초기화 하기 위함이지 않을까?
클래스명(전달값){초기화명령문}
```java
class Person {
    String name;
    int age;
    Person(String name, int age){
        this.name = name; // 초기화 작
        this.age = age;
    }
}
```

```java
public static void main(String[] args){
    Person person = new Person("이승희", 26);
}
```
Q1. 여기서 잠깐 생각을 해보자. 생성자는 왜 필요한 걸까?
1. 이 객체를 생성할 때, 처음부터 반드시 필요한 데이터가 있다. 뺴먹지 않고 처음부터 넣어줄 수 있다.
2. 인스턴스 변수가 여러개라면, .을 여러줄을 써야하지만, 생성자를 잘 정의하면 한번에 깔끔하게 할당 가능.
3. 만약 객체를 만든 후에 변수를 절대 건들면 안된다면, private로 변수를 설정하고, 생성자 정의하고, setter을 만들지 않으면, .으로 접근 못하니깐, 유일한 할당 방법은 객체 생성 시. 그래서 안전하다.

9. Getter와 Setter
Getter: 인스턴스의 변수 값을 반환.
```java
class Person {
    int age;
    public int getAge(){
        return age;
    }
}
```
Setter: 인스턴스 변수의 값 설정.
```java
class Person{
    private age;
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        }
    }
}
```

10. 접근 제어자
접근 권한 지정
클래스 앞, 인스턴스 변수 앞, 인스턴스 메서드 앞에 접근제어자 둘 수 있다.
private -> 해당 클래스 내에서만 접근 가능
public -> 모든 클래스에서
default (아무것도 적지 않았을 때) -> 같은 패키지 내에서만
protected -> 같은 패키지 내에서, 다른 패키지인 경우 자식 클래스에서.

```java
    }
```

```java

    }

class Student extends Person {
    @Override
    public void introduce(){
        System.out.println("학생입니다");
```

16. 참조
객체의 메모리 주소를 가리킨다.
기본 자료형은 실제 값을 저장하는 거고, 참조 자료형은 값이 저장된 메모리 주소를 저장한다.
String, Person, Student 같은 클래스들이 참조 자료형이다. 이는 값이 저장된 메모리 주소를 저장한다.
int a = 3; int b = 4; 일 때, a = b;를 한다면, a == 4, b == 4 즉, 값이 복사되는 건데,
만일 String s1 = "사과"; String s2 = "바나나"; 일 떄 s1 = s2는?
위와 같은 상황에는 s1와 s2는 s2가 가리키는 데이터 주소를 가리키게 된다.

17. Final
변경할 수 없게 하기 위해서 사용.
```java
class Person {
    final String name = "철수";
    public final void introduce(){
        System.out.println("사람입니다");
    }
}
```
위와 같은 경우에는 변수도 값을 바꿀 수 없고, 상속을 한 후에 메서드도 오버라이드 할 수 없다. 절대 바뀌어서는 안되는 중요한 변수나 메서드에 사용된다고 볼 수 있겠다.

## 추상클래스와 인터페이스
추상클래스는 아직 완성되지 않은 클래스
```java
abstract class Shape {
    abstract double calculateArea(); // 구체적으로 함수의 기능을 작성하지 않음.
}

```
```java
class Square extends Shape {
    pricate double s;
    public Square(double s) {
        this.s = s;
    }
    double calculateArea() {
        return s * s;
    }
}
```
```java
class Circle extends Shape {
    private double r;
    public Circle(double r){
        this.r = r;
    }
    double calculateAre(){
        return Math.PI * r * r;
    }
}
```

인터페이스는 클래스를 작성할 떄 기본이 되는 뼈대.
```java
interface Shape {
    double calculateArea();
}
```
이러면 인터페이스를 구현하는 클래스에서는 calculateArea()를 반드시 구현해야한다 -> 이 부분이 추상 클래스와의 차이점
```java
class Square implemnets Shape {
    private double s;
    public Square (double s) { 
        this.s = s;
    }
    double calculateArea(){
        return s * s;
    }
}
```

```java
class Circle implements Shape{
    private double r;
    Circle(double r){
        this.r = r;
    }
    double calculateArea(){
        return Math.PI * r * r;
    }
}
```
